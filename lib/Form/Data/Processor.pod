=head1 NAME 

Form::Data::Processor - yet another form data processor


=head1 DESCRIPTION

Actually it is just another form data validator,  which was inspired by
L<HTML::FormHandler>. It has similar syntax and similar features, except one: it
is B<NOT> a form renderer (but it is written using L<Moose>, so you can extend
it as you wish).


=head1 Form::Data::Processor vs HTML::FormHandler

The main goal was increasing validation speed with saving count of validate
features as possible.

Please look at benchmarks (C<benchmarks> folder in distro).

On my machine "Intel(R) Core(TM) i5-2520M CPU @ 2.50GHz, 8GB, Arch Linux
3.12.5-1-ARCH" it looks like:

    $> perl benchmarks/simple.pl
                 Rate Create hfh Create fdp
    Create hfh 80.6/s         --       -83%
    Create fdp  480/s       495%         --

         Rate FDP HFH
    FDP 306/s  -- -8%
    HFH 334/s  9%  --

    ----------------------------------------

    $> perl benchmarks/medium.pl
    1..1
                 Rate Create hfh Create fdp
    Create hfh 22.5/s         --       -44%
    Create fdp 39.9/s        78%         --
    ok 1 - FDP::result equals to HFH::values

          Rate  HFH  FDP
    HFH 38.7/s   -- -90%
    FDP  372/s 861%   --

    ----------------------------------------

    $> perl benchmarks/complex.pl 
    1..2
                 Rate Create fdp Create hfh
    Create fdp 5.85/s         --       -42%
    Create hfh 10.1/s        73%         --
    ok 1 - FDP::result equals to HFH::values

             Rate x1 HFH x1 FDP
    x1 HFH 29.3/s     --   -88%
    x1 FDP  247/s   745%     --
    ok 2 - FDP::result equals to HFH::values

              Rate x32 HFH x32 FDP
    x32 HFH 1.09/s      --    -95%
    x32 FDP 23.3/s   2027%      --
